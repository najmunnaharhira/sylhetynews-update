# Migration Guide: Moving from Mock Data to Firebase

## Overview

This guide helps you migrate existing mock news data to Firebase Firestore.

## Current State

Your project currently uses mock data in:
```
src/data/newsData.ts
```

This file contains hardcoded news articles. We'll migrate this to Firebase so articles are managed through the admin panel.

## Migration Steps

### Option 1: Manual Migration (Recommended for small datasets)

1. Go to Firebase Console > Firestore Database
2. Create categories first:
   - Click "Create collection" > Name it "categories"
   - Add documents for each category (Breaking News, Politics, Sports, etc.)
3. Create news collection:
   - Click "Create collection" > Name it "news"
   - Click "Add document" for each article
   - Fill in the fields matching the schema

### Option 2: Bulk Upload via Firebase Console

1. Export your `newsData.ts` to JSON format
2. Upload to Firestore using Firebase Console's import feature

### Option 3: Programmatic Migration

Create a migration script:

```typescript
// scripts/migrate-data.ts
import { db } from '../src/config/firebase';
import { collection, addDoc, Timestamp } from 'firebase/firestore';
import newsData from '../src/data/newsData';

async function migrateData() {
  try {
    // Create categories first
    const categories = ['Breaking News', 'Politics', 'Sports', 'Technology'];
    const categoryMap: { [key: string]: string } = {};

    for (const cat of categories) {
      const docRef = await addDoc(collection(db, 'categories'), {
        name: cat,
        slug: cat.toLowerCase().replace(/\s+/g, '-'),
      });
      categoryMap[cat] = docRef.id;
    }

    // Migrate articles
    for (const article of newsData) {
      await addDoc(collection(db, 'news'), {
        title: article.title,
        content: article.content,
        summary: article.summary || article.content.substring(0, 150),
        category: categoryMap['Breaking News'], // Map your categories
        author: article.author || 'Admin',
        imageUrl: article.imageUrl || '',
        featured: article.featured || false,
        published: true,
        tags: [],
        views: 0,
        createdAt: Timestamp.now(),
        updatedAt: Timestamp.now(),
      });
    }

    console.log('Migration completed successfully!');
  } catch (error) {
    console.error('Migration error:', error);
  }
}

migrateData();
```

Run with:
```bash
npx tsx scripts/migrate-data.ts
```

## Data Mapping

Map your existing data structure to Firebase:

### Old Structure (newsData.ts)
```typescript
{
  id: string;
  title: string;
  summary: string;
  content: string;
  author: string;
  imageUrl: string;
  date: string;
  featured?: boolean;
}
```

### New Structure (Firebase)
```typescript
{
  id: string;              // Auto-generated by Firestore
  title: string;           // Same as before
  summary: string;         // Same as before
  content: string;         // Same as before
  category: string;        // NEW - Category ID
  author: string;          // Same as before
  imageUrl: string;        // Same as before
  featured: boolean;       // NEW - Boolean instead of implicit
  published: boolean;      // NEW - Visibility control
  tags: string[];          // NEW - Empty array initially
  views: number;           // NEW - Track views
  createdAt: Timestamp;    // NEW - Auto timestamp
  updatedAt: Timestamp;    // NEW - Auto timestamp
}
```

## After Migration

### Update Frontend Components

Replace imports from mock data:

**Before:**
```typescript
import { mockNews } from '../data/newsData';

export default function HomePage() {
  return (
    <div>
      {mockNews.map(article => (...))}
    </div>
  );
}
```

**After:**
```typescript
import { useNews } from '../hooks/useNews';

export default function HomePage() {
  const { articles, loading } = useNews();
  
  if (loading) return <div>Loading...</div>;
  
  return (
    <div>
      {articles.map(article => (...))}
    </div>
  );
}
```

### Remove Mock Data File

Once migration is complete, you can remove:
```bash
rm src/data/newsData.ts
```

## Rollback Plan

If migration fails:

1. Firestore supports deleting collections
2. Go to Firebase Console > Firestore
3. Select collection > Delete collection
4. Try again or use manual method

## Performance Considerations

- Firestore has query limits (read carefully)
- Consider pagination for large datasets
- Use indexes for complex queries
- Monitor read/write operations in Firebase Console

## Cost Implications

Firebase free tier includes:
- 50,000 reads/day
- 20,000 writes/day
- 20,000 deletes/day
- 1GB storage

For small to medium news sites, this is usually sufficient.
